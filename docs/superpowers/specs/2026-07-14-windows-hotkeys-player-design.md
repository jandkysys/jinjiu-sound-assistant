# Windows 全局快捷键与分板块播放器设计

## 目标

在金玖音效助手 1.0.4 中统一 Electron 全局快捷键状态、音效与功能快捷键冲突规则，完善主播音效/背景音乐播放互斥，缩小悬浮窗，增加完整 PC 模拟键盘，并在真实 Windows Electron 环境验收后再打包。

## 快捷键架构

- Electron 主进程是快捷键启用状态的唯一真源。
- 每次进程启动均将总开关设为关闭；音效快捷键和功能快捷键都不注册。
- 用户从主界面复选框或悬浮 Toggle 开启后，主进程注册两类快捷键并广播状态；任意界面收到广播后立即同步。
- 关闭时调用 Windows `globalShortcut.unregister` 注销所有动态音效与功能绑定，但保留绑定配置。
- 音效与功能快捷键共用标准化后的按键标识和冲突表。同一个按键只能有一个动作；用户确认替换时先清除旧功能绑定，再保存新音效绑定。
- 支持 F1-F12、主键盘、导航区、数字小键盘 0-9、小数点和加减乘除；数字小键盘 Enter 不提供绑定。主键盘 `1` 与 `Num1` 使用不同标识。

## 播放架构

- 主播音效属于 host scope，背景音乐属于 bgm scope；每个 scope 同时只允许一个音频实例，两类可以并行。
- 鼠标或快捷键再次触发同一条音效/音乐时，一律将 `currentTime` 归零并从头播放。
- 三个界面板块分别记录本板块最近触发的内容。切换板块只改变播放器显示，不停止其他板块音频；无记录显示“暂无播放”。
- 主播音效循环开关持久化；新主播音效触发时停止上一条循环。

## 界面

- 悬浮窗内容约 150×78，删除快捷键数量灰色栏和多余空白，保留左键拖动及位置持久化。
- 主窗口初始尺寸约 1280×820，保留系统边框与自由缩放。
- 模拟键盘使用完整 PC 布局。已绑定键使用现有一级/二级分类风格的高对比色、边框和文字。

- 仅在可自动验证项目通过且可完成的桌面验收通过后运行 electron-builder；无法自动模拟的项目明确标记为人工待验收，不虚报。

## 明确取舍

- 快捷键绑定、循环状态和悬浮窗位置保留；快捷键启用状态不跨进程保留，每次启动固定关闭。
- 不改云端同步、登录权限、题词器或音效数据结构之外的功能。
